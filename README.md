 11. 최종 결론
Make와 Zapier에서 같은 회원 이메일 자동화를 만들고 직접 실행하였다. 두 도구 모두 새로운 회원 신청을 확인하고 일반회원과 VIP회원을 나눈 뒤 알맞은 이메일을 보낼 수 있었다.
Zapier는 단계가 위에서 아래로 보여 처음 사용할 때 순서를 이해하기 쉬웠다. Make는 전체 자동화가 그림처럼 보여 두 경로가 어떻게 연결되었는지 확인하기 쉬웠다. 여러 조건과 기능을 계속 추가한다면 Make가 더 편리하고, 간단한 자동화를 빠르게 만들 때는 Zapier가 더 적합하다고 판단하였다.
이번 프로젝트를 통해 Trigger는 자동화를 시작하는 신호이고, Action은 자동으로 실행되는 일이라는 것을 알게 되었다. Router, Filter, Paths는 입력된 정보를 확인하고 알맞은 길로 보내는 역할을 한다는 것도 직접 확인하였다.
