1. 브랜드 아이덴티티

브랜드명:멜로숲 뮤직

분야:유아 음악 놀이 교육 서비스

타겟:3~7세 유아 및 학부모

톤앤매너:따뜻한 숲속 분위기의 음악 교실, 귀여운 3D 애니메이션, 밝고 안전한 교육 콘텐츠

차별점(USP):AI 음악 친구와 함께 리듬, 악기, 소리 표현을 놀이처럼 경험하는 유아 음악 교육 콘텐츠

광고 목적:브랜드 인지도 향상

핵심 메시지:“AI 친구와 시작하는 음악놀이, 멜로숲 뮤직”



2. 기술적 설명:

이번 광고 영상은 텍스트 프롬프트를 기반으로 이미지, 영상, 오디오를 각각 생성형 AI로 제작한 뒤 편집 도구에서 하나의 광고 영상으로 통합하였다.

제작 과정:

① 기획 및 스토리보드 작성
• 브랜드 아이덴티티 설정
• 타겟 및 핵심 메시지 정의
• 10초 이내 광고를 위해 4개의 짧은 씬으로 구성
• 씬별 목표 메시지, 화면 구성, 화면 카피 설정

② 이미지 생성
씬별 핵심 장면은 ChatGPT의 이미지 생성 기능을 활용하여 제작하였다. 학습 네이토의 이미지 생성 기능에도 동일한 프롬프트를 입력하여 결과를 비교했으나, 캐릭터의 형태와 3D 스타일이 기획 의도와 다르게 생성되어 최종 이미지로 사용하지 않았다.
ChatGPT에서는 다음 요소를 중심으로 씬별 기준 이미지를 생성하였다.
• 여자아이 캐릭터
• AI 음악 친구
• 따뜻한 숲속 음악 교실
• 귀여운 3D 애니메이션 스타일
캐릭터의 외형, 색상, 배경 분위기와 애니메이션 스타일을 유사하게 유지하여 씬 사이의 시각적 일관성을 확보하였다. 최종적으로 기획 의도에 가까운 결과를 생성한 ChatGPT 이미지를 비디오 제작의 기준 이미지로 사용하였다.
③ 비디오 생성
생성한 이미지를 기준 이미지로 활용하여 짧은 애니메이션 영상을 제작하였다.
• 악기 연주 움직임
• 떠다니는 음표
• 부드러운 카메라 이동
• 자연스러운 캐릭터 움직임
• 반짝이는 빛과 음표 애니메이션
영상 생성 과정에서는 잘못된 글자가 생성되는 것을 방지하기 위해 텍스트와 로고를 넣지 않았다. 화면 카피와 브랜드명은 CapCut 편집 단계에서 별도로 삽입하였다.
캐릭터와 스타일의 일관성을 유지하기 위해 씬별로 생성한 3D 이미지를 Gemini의 기준 이미지로 업로드하였다. 화면 비율은 16:9, 스타일은 따뜻한 색감의 비사실적 3D 애니메이션으로 설정하였다. 프롬프트에는 캐릭터와 악기의 형태를 유지하고 불필요한 사물이나 텍스트를 생성하지 않도록 제한 조건을 추가하였다.
④ 오디오 생성
생성형 AI Suno를 활용하여 밝고 따뜻한 유아 광고용 배경음악을 제작하였다. 실로폰, 우쿨렐레, 탬버린, 부드러운 손뼉 소리의 분위기가 느껴지도록 프롬프트를 구성하였다.
동일한 기획 방향을 바탕으로 BGM 후보 3곡을 생성한 뒤, 멜로숲 뮤직의 밝고 따뜻한 분위기와 9.6초 광고 흐름에 가장 잘 어울리는 1곡을 최종 선정하였다.

⑤ 통합 편집
CapCut을 이용하여 다음 작업을 진행하였다.
• 씬별 컷 편집
• 화면 카피 삽입
• 브랜드명 삽입
• 영상과 음악의 타이밍 조정
• 오디오 레벨 조정
• 최종 MP4 파일 출력

CapCut은 핵심 비주얼과 오디오를 만드는 용도가 아니라 생성형 AI로 제작한 결과물을 하나의 광고 영상으로 통합하는 용도로 사용하였다.

화면 비율 변경 과정
초기에는 모바일 숏폼 콘텐츠를 고려하여 9:16 세로형으로 이미지를 제작하였다. 이후 과제 제출 규격이16:9 가로형 광고임을 확인하고 화면 비율을 변경하였다.
세로 이미지를 단순히 확대하거나 잘라내면 여자아이 캐릭터와 악기가 잘릴 가능성이 있었다. 따라서 기존 캐릭터와 3D 애니메이션 스타일을 유지하면서 좌우 배경을 자연스럽게 확장한 16:9 이미지를 다시 생성하였다.
수정된 이미지를 기준으로 16:9 애니메이션 영상을 제작하여 광고 규격과 주요 피사체의 구도를 모두 유지하였다.

ChatGPT에서는 여자아이, AI 음악 친구, 악기와 따뜻한 음악 교실의 분위기가 기획 의도에 비교적 가깝게 생성되었다. 또한 이미지의 수정 방향을 대화형으로 이어가기 편리하여 최종 이미지 생성 도구로 ChatGPT를 선택하였다.

Sora 2는 3D 애니메이션의 움직임과 영상 표현이 풍부했지만, 기준 이미지와 비교했을 때 캐릭터의 외형이나 악기 연주 동작이 달라지는 경우가 있었다. 특히 원본 이미지의 캐릭터와 구도를 동일하게 유지하는 데 어려움이 있었다.
Gemini는 업로드한 이미지를 시각적 기준으로 활용하여 캐릭터, 악기와 배경의 형태를 비교적 안정적으로 유지하였다. 움직임을 구체적으로 제한했을 때 악기 연주와 음표 애니메이션도 기획 의도에 가깝게 표현되었다.
